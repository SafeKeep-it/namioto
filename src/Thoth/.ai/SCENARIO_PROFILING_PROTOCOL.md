# Scenario Profiling Protocol (Canonical)

## Purpose

Prevent mixed-dataset analysis and conflicting hotspot narratives.

## Canonical Inputs

- Wall-clock ranking: `tmp/per_scenario_bench_output.csv` generated by `tmp/per_scenario_bench.sh`.
- CPU hotspots: only `tmp/scenario-*-cpu.speedscope.json`.
- Memory snapshots: `tmp/scenario-*-t5.report.txt` and `tmp/scenario-*-t15.report.txt`.

## Explicitly Ignored Inputs

- Legacy CPU traces named `tmp/scenario-*.speedscope.json` that do not end with `-cpu.speedscope.json`.
- Mixed comparisons between full-run traces and per-scenario isolated traces.

## Normalization Rules

- Scenario ranking authority: wall-clock output (`real_ms`) from benchmark script.
- CPU method comparisons are supplemental and interpreted inside scenario context.
- Per-frame normalization uses exact per-scenario frame count from run config.
- Do not divide by scenario count.

## Regeneration Steps

1. `bash tmp/per_scenario_bench.sh > tmp/per_scenario_bench_output.csv`
2. `python3 tmp/build_canonical_scenario_summary.py`
3. Consume only:
   - `tmp/scenario_isolated_summary.md`
   - `tmp/scenario_isolated_summary.json`
